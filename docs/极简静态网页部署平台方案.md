# 极简静态网页部署平台方案

## 1. 项目定位

本项目是一个非常简单的静态网页部署 Demo。

当前版本只支持上传**单个 HTML 文件**。用户填写网页名字，选择一个 `.html` 文件，平台保存后在首页生成项目卡片。点击卡片时，首页打开一个大预览窗口，用 `iframe` 直接显示这个网页。

最小流程：

```text
用户上传 HTML 文件 -> 平台保存为 index.html -> 首页生成项目卡片 -> 点击卡片大窗口预览网页
```

第一版不需要用户登录，也不需要填写用户名。

## 2. 第一版功能

第一版只做这些功能：

```text
一个平台首页
一个右上角上传按钮
一个上传弹窗
一个项目卡片列表
一个项目预览大窗口
一个项目访问地址
```

用户可以：

```text
填写网页名字
上传单个 .html 文件
在首页看到上传后的项目卡片
点击项目卡片，在大预览窗口里直接查看网页
点击“在新页面打开”，直接访问 /site/{id}
复制 /site/{id} 地址分享给别人
```

第一版暂时不做：

```text
注册登录
用户名输入
权限管理
项目编辑
项目删除
上传 zip
上传文件夹
多文件项目上传
数据库
自定义域名
访问统计
版本回滚
在线代码编辑器
前端框架
```

## 3. 页面形态

整个平台只有一个主要页面，也就是首页 `/`。

页面结构：

```text
首页
  顶部栏
    左侧：平台名称，例如 HtmlDeploy
    右侧：上传网页按钮

  内容区
    项目卡片列表
      卡片：网页名字
      卡片：访问地址
      卡片：预览项目按钮
      卡片：在新页面打开按钮

  上传弹窗
    网页名字输入框
    HTML 文件选择控件
    取消按钮
    确定上传按钮

  项目预览大窗口
    顶部：网页名字、访问地址、在新页面打开按钮、关闭按钮
    内容：iframe 显示 /site/{id}
```

上传控件使用普通单文件选择：

```html
<input type="file" accept=".html,text/html">
```

这样文件选择窗口中会直接显示 `.html` 文件。

## 4. 上传流程

完整流程：

```text
用户打开首页
  ↓
点击右上角“上传网页”
  ↓
填写网页名字
  ↓
选择一个 .html 文件
  ↓
点击“确定上传”
  ↓
后端检查文件是否为 HTML
  ↓
后端生成随机项目 ID
  ↓
检查项目 ID 是否重复
  ↓
后端创建 storage/sites/{id}/ 目录
  ↓
后端把上传文件保存为 storage/sites/{id}/index.html
  ↓
把项目信息写入 data/sites.json
  ↓
首页刷新项目卡片列表
  ↓
新项目卡片出现在首页
  ↓
用户点击卡片，在大预览窗口里查看网页
```

上传示例：

```text
网页名字：我的小游戏
上传文件：game.html
```

保存后：

```text
项目 ID：abc123
项目地址：/site/abc123
文件位置：storage/sites/abc123/index.html
```

首页卡片展示：

```text
我的小游戏
访问地址：/site/abc123
[预览项目] [在新页面打开]
```

## 5. 技术方案

第一版使用最简单的技术栈：

```text
Node.js
Express
Multer
本地 JSON 文件
本地文件夹存储
单 HTML 文件前端
```

说明：

```text
Express
  提供首页、接口和 HTML 文件访问路由。

Multer
  处理单个 HTML 文件上传。

data/sites.json
  保存项目列表。

storage/sites/{id}/index.html
  保存用户上传的 HTML 文件。

public/index.html
  平台首页。第一版只写这一个 HTML 文件。
```

平台首页前端要求：

```text
只创建 public/index.html
HTML 写在 index.html 里
CSS 写在 <style> 标签里
JavaScript 写在 <script> 标签里
不单独创建 public/style.css
不单独创建 public/app.js
不引入 React、Vue 等前端框架
```

## 7. 数据结构

`data/sites.json` 保存项目列表：

```json
[
  {
    "id": "abc123",
    "title": "我的小游戏",
    "createdAt": "2026-06-02T10:00:00.000Z"
  }
]
```

项目访问地址：

```text
/site/{id}
```

例如：

```text
/site/abc123
```

## 8. 项目 ID 唯一性

项目 ID 必须检查不重复，避免新项目覆盖旧项目。

规则：

```text
生成随机 ID
  ↓
检查 data/sites.json 是否已有相同 id
  ↓
检查 storage/sites/{id}/ 目录是否已经存在
  ↓
如果任意一处冲突，就重新生成 ID
  ↓
直到得到一个未使用的 ID
```

## 9. 后端接口

### GET /

返回平台首页：

```text
public/index.html
```

### GET /api/sites

返回项目列表。

### POST /api/sites

上传单个 HTML 文件。

表单字段：

```text
title
file
```

处理逻辑：

```text
检查 title 是否为空
检查是否上传了文件
检查文件扩展名是否为 .html 或 .htm
生成不重复的项目 ID
创建 storage/sites/{id}/ 目录
将上传文件保存为 storage/sites/{id}/index.html
把项目记录写入 data/sites.json
返回项目 ID、标题和访问地址
```

返回示例：

```json
{
  "id": "abc123",
  "title": "我的小游戏",
  "url": "/site/abc123"
}
```

### GET /site/:id

访问用户上传的网页。

处理逻辑：

```text
查找 storage/sites/{id}/index.html
如果文件存在，返回 index.html
如果文件不存在，返回 404
```

## 10. 文件上传规则

当前版本只支持：

```text
单个 HTML 文件
.html 或 .htm 扩展名
上传后统一保存为 index.html
```

当前版本不支持：

```text
多个文件
文件夹
CSS/JS/图片单独上传
zip
服务端构建
```

如果网页需要 CSS 和 JavaScript，建议先写在同一个 HTML 文件中：

```html
<style>
  /* CSS */
</style>

<script>
  // JavaScript
</script>
```

## 11. 预览窗口逻辑

首页项目卡片点击后：

```text
读取项目 url，例如 /site/abc123
打开大预览窗口
设置 iframe.src = /site/abc123
iframe 显示用户上传的 HTML 网页
```

预览窗口顶部需要有：

```text
网页名字
访问地址
在新页面打开按钮
关闭按钮
```

## 12. 完成标准

第一版 Demo 完成标准：

```text
平台首页只有 public/index.html 一个前端文件
首页右上角有上传按钮
点击上传按钮打开上传弹窗
上传弹窗只需要填写网页名字和选择一个 HTML 文件
非 HTML 文件会被拒绝
上传后首页出现项目卡片
项目 ID 不会和已有项目重复
点击项目卡片打开大预览窗口
大预览窗口通过 iframe 显示 /site/{id}
预览窗口可以关闭
预览窗口可以在新页面打开 /site/{id}
刷新首页后，项目卡片仍然存在
```

## 13. 一句话总结

这个 Demo 是一个极简静态 HTML 页面部署平台：

```text
用户只填写网页名字并上传一个 HTML 文件。平台把文件保存为 storage/sites/{id}/index.html，生成不重复的项目 ID，在首页展示项目卡片。点击卡片时，首页打开大预览窗口，用 iframe 显示 /site/{id}。
```
