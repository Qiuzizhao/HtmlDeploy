# 学生名单紧凑卡片设计

## 目标

学生名单不再使用整行编辑表单，改为与后台“违禁词”列表一致的紧凑卡片网格，提升浏览效率并减少横向占用。

## 展示与交互

- 名单容器复用违禁词列表的自适应网格、间距、边框、圆角和浅色背景。
- 每名学生显示为白色紧凑卡片，默认只显示选择框和学生姓名。
- 单击学生卡片后，仅在该卡片上浮出“删除”按钮；单击其他位置或按 `Escape` 收起。
- 删除姓名输入框、班级下拉框、创建时间和“保存”按钮；学生改名及转班不在列表内提供。
- 保留班级筛选、姓名搜索、新增学生弹窗、批量选择和删除选中。
- 单卡删除继续调用现有删除接口和确认流程；勾选继续驱动“删除选中”按钮状态。
- 窄屏继续使用自适应卡片网格，不恢复纵向编辑表单。

## 范围

只修改 `public/admin.html` 的学生列表样式和 `renderStudents()` 输出，并在 `test/app.test.js` 添加布局回归测试。不修改数据库、API、导入、搜索或删除逻辑。

## 验收标准

- 学生名单视觉结构与违禁词卡片网格一致。
- 每张学生卡默认只显示选择框和姓名，单击后才显示删除按钮。
- 页面不再为每名学生渲染输入框、班级下拉、创建时间或保存按钮。
- 单删、批量选择与批量删除仍可使用。
