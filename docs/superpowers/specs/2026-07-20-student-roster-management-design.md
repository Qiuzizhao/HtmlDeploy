# 学生名单管理设计

## 目标

在后台左侧栏新增“学生名单”功能，让管理员可以为现有班级维护学生姓名，并通过 Excel、CSV 或逐行粘贴批量导入。该名单为未来的班级专属登录页面提供基础数据，但本次不实现学生登录，也不改变现有项目上传和作者署名流程。

## 范围

本次包含：

- 按班级查看学生名单和人数。
- 按姓名搜索学生。
- 手动新增单个学生。
- 上传 `.xlsx`、`.csv` 名单。
- 粘贴每行一个姓名的名单。
- 导入预览、重复识别和确认写入。
- 修改学生姓名、转移班级、单个删除和批量删除。
- 后台权限保护、数据校验和自动化测试。

本次不包含：

- 学生登录页面或学生会话。
- 对外公开学生名单接口。
- 上传项目时自动填写作者。
- 学号、密码或其他学生个人信息。

## 数据结构

新增 SQLite 表 `students`：

- `id TEXT PRIMARY KEY`
- `class_id TEXT NOT NULL`
- `name TEXT NOT NULL`
- `created_at TEXT NOT NULL`
- `updated_at TEXT`
- `FOREIGN KEY (class_id) REFERENCES classes(id)`
- `UNIQUE (class_id, name)`

为 `class_id` 和 `name` 建立查询索引。学生只保存姓名和所属班级；不同班级允许同名，同一班级不允许同名。

姓名写入前必须去除首尾空白，并将连续空白合并为一个空格。空姓名拒绝保存，姓名最大长度为 40 个字符。

删除仍包含学生的班级时拒绝操作，并提示管理员先删除或转移学生，避免名单被级联误删。

## 后台界面

后台左侧栏在“班级管理”下方新增“学生名单”，对应独立后台视图。

页面顶部提供：

- 班级选择器。
- 当前班级学生总数。
- 姓名搜索框。
- 单个学生姓名输入和新增按钮。

批量导入区域提供：

- `.xlsx`、`.csv` 文件选择。
- 每行一个姓名的粘贴文本框。
- “解析名单”按钮。
- 导入预览和“确认导入”按钮。

学生列表提供多选框、姓名、所属班级、创建时间和操作按钮。单项操作包括改名、转班、删除；批量操作包括批量删除。删除操作必须二次确认。

## 名单解析与导入

导入前必须先选择目标班级。

Excel 和 CSV 优先识别表头为“姓名”的列；没有该表头时，使用第一个包含非空值的列。纯文本按行拆分。所有来源都经过同一姓名规范化和校验流程。

解析结果分为：

- 有效且可新增的姓名。
- 导入内容内部的重复姓名。
- 数据库中已存在的姓名。
- 空白、超长或无法识别的无效行。

管理员在预览后确认写入。已有数据不覆盖，重复数据自动跳过。完成后明确显示新增人数、名单内重复数、数据库已有数和无效行数。

## 管理规则

- 新增或改名与当前班级已有姓名冲突时，返回明确错误。
- 转班时目标班级已有同名学生则拒绝转移。
- 单个编辑只提交发生变化的姓名或班级。
- 搜索限制在当前所选班级内。
- 批量删除只作用于当前已选择的学生 ID，并要求确认。
- 删除、转班或改名成功后重新加载当前班级名单和统计。

## 服务端接口

新增后台专用接口，全部经过现有 `requireAdmin` 中间件：

- `GET /api/admin/students?classId=<id>&q=<name>`：读取班级名单。
- `POST /api/admin/students`：新增单个学生。
- `POST /api/admin/students/import`：批量导入已解析的姓名数组。
- `PUT /api/admin/students/:id`：改名或转班。
- `DELETE /api/admin/students/:id`：删除单个学生。
- `POST /api/admin/students/bulk-delete`：批量删除学生。

文件解析在后台页面完成，服务端只接收经过预览确认的姓名数组，并再次执行完整校验和去重。这样服务端接口保持简单，仍以数据库约束作为最终一致性保障。

## 代码边界

- `src/db/schema.js`：创建学生表和索引。
- `src/db/runtime-store.js`：学生记录的规范化、查询和原子写入。
- `src/db/repositories/students.js`：暴露学生仓库接口。
- `src/db/repositories/index.js`：注册学生仓库。
- `src/app.js`：权限保护、请求校验和学生 API。
- `public/admin.html`：学生名单视图、Excel/CSV/文本解析和管理交互。
- `test/app.test.js`：数据库、API、权限和页面回归测试。

Excel 解析使用轻量的浏览器端 SheetJS 构建产物并随项目本地提供，不依赖运行时外部 CDN；CSV 和文本使用页面内解析逻辑。生产环境断网时仍可使用所有导入功能。

## 错误处理与安全

- 未登录后台访问学生 API 返回 `401`。
- 班级不存在返回 `404`。
- 姓名为空、超长或请求结构错误返回 `400`。
- 同班同名冲突返回 `409`。
- 导入数量设置合理上限，单次最多 2,000 个有效姓名，超出时拒绝请求。
- 服务端不信任浏览器解析结果，重新规范化、校验和去重。
- 本次不提供任何公开学生查询接口，公开首页不能读取学生数据。

## 测试与验收

自动化测试覆盖：

- 学生表创建、同班唯一约束和跨班同名。
- 未授权访问学生接口被拒绝。
- 新增、查询、搜索、改名、转班、单删和批量删除。
- 不存在班级、无效姓名、同名冲突和导入上限。
- 导入统计中新增、内部重复、已有和无效数量准确。
- 含学生班级不可删除。
- 后台左侧栏、学生名单视图和三种导入入口存在。
- 现有班级管理、作品上传和作者输入行为保持不变。

人工验收：选择现有班级，分别用 Excel、CSV 和粘贴名单导入；确认预览统计准确，写入后可搜索、改名、转班和删除，并确认刷新页面后数据仍在。
