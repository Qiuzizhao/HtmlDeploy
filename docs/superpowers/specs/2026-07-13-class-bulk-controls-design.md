# 班级批量开关设计

## 目标

在后台“班级管理”的班级列表上方提供四个明确的快捷按钮：启用全部上传、关闭全部上传、启用全部密码、解除全部密码。

## 方案

新增两个仅管理员可调用的批量接口，分别统一设置所有班级的 `uploadEnabled` 与 `passwordEnabled`。接口一次读取并一次写回完整班级列表，避免浏览器逐班请求造成部分成功、部分失败。

后台在班级列表上方增加“批量操作”工具栏。点击按钮后先确认影响范围；确认后禁用批量按钮并显示处理中状态，请求成功后用接口返回的班级列表整体替换本地状态并重新渲染，失败时保持原状态并展示错误。

## 接口

- `PATCH /api/classes/upload-enabled`，请求体 `{ "uploadEnabled": boolean }`
- `PATCH /api/classes/password-enabled`，请求体 `{ "passwordEnabled": boolean }`

成功返回更新后的完整班级数组。未登录返回 401。请求值不是布尔值时返回 400。空班级列表返回空数组。

批量路由必须注册在 `/:id/...` 路由之前，避免固定路径被识别为班级 ID。

## 界面与反馈

四个按钮位于新增班级表单与班级列表之间，并沿用现有按钮样式。没有班级时全部禁用。操作期间四个按钮统一禁用，防止并发覆盖。

确认文案与成功提示明确指出操作对象是“所有班级”。单班级开关保持不变。

## 测试

- 接口可一次关闭及启用所有班级上传。
- 接口可一次解除及启用所有班级密码。
- 非布尔请求被拒绝，未登录请求被拒绝，空列表安全返回。
- 后台 HTML 包含四个按钮、批量处理函数、接口调用与事件绑定。
