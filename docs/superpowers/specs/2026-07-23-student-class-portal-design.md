# 学生专属班级页设计

## 目标

教师为每个班级启用一个不可猜测的专属链接。学生打开链接后看到本班学生姓名气泡，点击自己的姓名即可进入项目站。进入后服务器锁定学生身份、上传班级和作者姓名，学生不能修改作者或把作品上传到其他班级。

## 后台设置

学生名单页随当前班级显示专属页设置：

- “启用专属页”控制链接是否可访问，默认关闭。
- “开放全部作品”控制该班学生是否可浏览全部班级，默认关闭。
- “复制专属链接”复制 `/class/<随机令牌>` 完整地址。
- “重置链接”生成新令牌并立即使旧链接与旧学生会话失效。

随机令牌只在管理员接口返回，不进入公共班级接口。

## 学生入口

`/class/<随机令牌>` 展示班级名称、简短提示和本班姓名气泡。页面只返回该班学生的 ID 与姓名，不包含其他班级、密码或后台字段。点击姓名向服务端创建学生会话并跳转到首页。

学生会话使用签名且 `HttpOnly` 的 Cookie，保存学生 ID、班级 ID、专属页令牌版本，最长 24 小时。学生被删除、转班、专属页停用或链接重置后，会话自动失效。

## 首页身份与浏览权限

首页启动时读取当前学生会话。有效会话在右上角显示“班级名 · 学生姓名”和“退出”。退出清除会话并返回所属班级的姓名页。

- 未开放全部作品：只返回和显示学生所属班级及该班作品。
- 已开放全部作品：保留全部作品与各班浏览入口。
- 浏览范围不影响上传目标；上传始终绑定学生所属班级。

## 上传约束

学生会话存在时：

- 上传弹窗以只读文本显示固定班级和作者。
- 前端不允许选择班级或编辑作者。
- `/api/sites` 和 `/api/upload-ai-name` 在服务端解析学生会话，忽略客户端的 `classId` 与 `author`，使用会话中的班级和数据库中的最新学生姓名。
- 仍执行班级上传开关、HTML 校验、重复项目和违禁词校验。

普通访客和管理员的原有浏览、班级密码与上传流程保持不变。

## 数据模型

`classes` 新增：

- `student_portal_enabled INTEGER NOT NULL DEFAULT 0`
- `student_portal_all_works INTEGER NOT NULL DEFAULT 0`
- `student_portal_token TEXT NOT NULL DEFAULT ''`

数据库架构版本升至 4，并对现有数据库执行幂等列迁移。

## 接口

- `GET /api/student-portal/:token`：返回启用班级和学生气泡数据。
- `POST /api/student-login`：校验令牌与学生归属，创建签名会话。
- `GET /api/student-session`：返回当前身份和浏览权限。
- `POST /api/student-logout`：清除会话。
- `PATCH /api/admin/classes/:id/student-portal`：更新启用及浏览范围。
- `POST /api/admin/classes/:id/student-portal/reset-link`：重置链接。

## 验收标准

- 教师能启用、配置、复制及重置每个班级的专属页。
- 学生通过姓名气泡直接登录，不输入密码。
- 首页右上角显示班级与姓名，并提供退出。
- 未开放全部作品时无法通过 UI 或接口读取其他班作品。
- 学生上传的班级和作者均由服务器强制确定，篡改请求无效。
- 旧链接、失效学生和被重置的会话均不能继续使用。
