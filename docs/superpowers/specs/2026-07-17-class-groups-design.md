# 班级分组设计

## 目标

后台支持创建、改名、删除和排序班级分组，并可在组内或跨组调整班级顺序。前台按相同分组和顺序显示班级按钮。

## 数据模型

新增 `class_groups` 表，字段包括 `id`、`name`、`position`、`created_at` 和 `updated_at`。`classes` 表新增 `group_id`，空字符串表示系统“未分组”。现有班级迁移后全部进入未分组，不修改密码、上传状态、作品或班级顺序。

“未分组”不是数据库中的普通分组，不允许改名或删除。普通分组删除后，其班级的 `group_id` 统一清空。

## 接口

- `GET /api/class-groups`：公开返回分组，供前后台渲染。
- `POST /api/class-groups`：管理员新增分组。
- `PUT /api/class-groups/order`：管理员保存分组顺序。
- `PUT /api/class-groups/:id`：管理员修改名称。
- `DELETE /api/class-groups/:id`：管理员删除分组并把班级移到未分组。
- `PUT /api/classes/order`：扩展为接收有序的 `{ id, groupId }` 项目，保存组内及跨组顺序；保留旧 `classIds` 格式兼容。

新增和编辑班级时校验 `groupId`；空字符串合法，非空值必须对应已有分组。

## 后台交互

班级管理面板增加分组工具栏和分组容器。管理员可以新增分组、修改名称、删除分组、拖动分组排序；班级行可在组内排序或拖到另一个分组。新增班级表单增加所属分组选择。

删除非空分组前明确提示班级会移到未分组。保存失败时重新加载服务器状态，避免页面顺序与数据库不一致。

## 前台展示

“全部作品”按钮固定在最前。其后按分组顺序显示非空分组，每组显示标题和班级按钮；未分组班级统一显示在“未分组”标题下。所有分组默认展开，空普通分组不显示。

## 测试与部署

测试覆盖旧数据库迁移、分组增删改排序、删除后班级归入未分组、跨组排序持久化，以及前台分组渲染。部署前备份 `data` 和 `storage`，重启后校验新表、新字段、PM2 和公网健康状态。
