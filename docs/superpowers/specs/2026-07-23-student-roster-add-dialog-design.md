# 学生名单新增弹窗与工具栏布局设计

## 目标

调整后台“学生名单”页面的操作布局：将“新增学生”和“删除选中”两个按钮放到学生搜索框右侧并横向排列；将原来的单人新增表单和页面内批量导入面板合并为一个“新增学生”弹窗，使管理员可以通过 Excel、CSV 或逐行粘贴一次新增多名学生。

## 页面布局

学生名单主工具栏按以下顺序排列：

1. 班级选择器。
2. 学生搜索框。
3. “新增学生”主按钮。
4. “删除选中”危险按钮。
5. 当前学生人数。

桌面端保持一行横向排列，搜索框占据剩余宽度，两个操作按钮紧邻搜索框右侧。按钮文字保持单行显示，不出现截图中的上下堆叠。

窄屏下工具栏允许自然换行；班级选择器和搜索框优先占满可用宽度，两个按钮仍保持横向相邻，不拆成上下两个独立按钮。

删除页面中的 `addStudentForm` 单人新增区域和原有内嵌“批量导入”面板，避免重复入口。

## 新增学生弹窗

点击“新增学生”打开居中模态对话框。弹窗包含：

- 标题“新增学生”。
- 关闭按钮。
- 当前目标班级提示。
- Excel/CSV 文件选择器。
- 多行粘贴框，提示“每行一个学生姓名”。
- “解析名单”按钮。
- 解析预览区域。
- “取消”按钮。
- “确认新增”主按钮。

弹窗复用当前页面已经上线的名单解析、姓名规范化、重复统计、无效行统计和服务端导入接口。Excel、CSV 与粘贴内容可以同时提供，解析时合并为一份名单。

“确认新增”只有在以下条件同时满足时才可点击：

- 已选择有效班级。
- 已成功解析出至少一名有效学生。
- 当前没有解析或导入请求正在执行。

## 交互流程

1. 管理员选择班级后点击“新增学生”。
2. 弹窗显示当前班级名称，管理员上传 Excel/CSV、粘贴姓名，或同时使用两种输入。
3. 点击“解析名单”后显示有效姓名、名单内重复数和无效行数，并列出姓名预览。
4. 点击“确认新增”后调用现有批量导入接口。
5. 成功时关闭弹窗，清空弹窗输入，刷新当前班级名单，并在主页面显示新增、重复、已有和无效数量。
6. 失败时弹窗保持打开，保留输入和预览，显示服务端错误，方便管理员修正后重试。

关闭方式包括右上角关闭按钮、取消按钮和点击遮罩层。解析或导入进行中时禁止关闭，避免隐藏仍在运行的任务。按 `Escape` 关闭空闲弹窗。

任何主动关闭操作都会清空未提交的文件、粘贴内容和解析预览；下次打开弹窗从空白状态开始。

## 状态与并发安全

现有 `studentImportGeneration` 和 `studentImportsInFlight` 状态继续作为唯一的解析与导入并发保护来源。

- 文件或粘贴内容变化时使旧解析结果失效。
- 旧解析完成时不能覆盖新输入。
- 导入进行中不能重新启用确认按钮。
- 旧请求完成时不能清空后来输入的新名单。
- 切换班级后旧班级请求不能改写当前班级列表或消息。

弹窗打开时锁定目标班级为当前选择；弹窗打开期间主页面班级选择器不可操作。关闭弹窗后恢复班级选择器。

## 删除选中

“删除选中”按钮仅改变位置，不改变现有行为：

- 未选择学生时保持禁用。
- 点击后显示确认提示。
- 只删除当前选中的学生 ID。
- 请求期间使用现有加载状态，完成后刷新名单。

## 可访问性

- 弹窗使用 `role="dialog"`、`aria-modal="true"` 和标题关联。
- 打开弹窗后焦点进入文件选择器或粘贴框。
- 关闭弹窗后焦点返回“新增学生”按钮。
- 弹窗打开时阻止背景滚动。
- 所有按钮保留清晰的禁用和加载状态。

## 代码范围

- `public/admin.html`：工具栏结构、弹窗结构和样式、导入状态复用、打开/关闭与焦点管理、事件绑定。
- `test/app.test.js`：工具栏顺序、弹窗结构、无重复内嵌入口、打开/关闭、导入成功/失败和并发状态回归测试。

本次不修改数据库、学生 API、Excel 解析依赖、学生登录、项目上传或作者署名逻辑。

## 验收标准

- 桌面端“新增学生”和“删除选中”位于搜索框右侧且横向排列。
- 页面不再显示单人姓名输入框或独立批量导入面板。
- 点击“新增学生”弹出可上传 Excel、CSV 和粘贴多行姓名的对话框。
- 解析预览和导入统计与当前功能一致。
- 成功后弹窗关闭并刷新名单；失败后保留输入。
- 弹窗空闲时可通过关闭按钮、取消、遮罩或 `Escape` 关闭。
- 解析或导入运行时不能关闭弹窗。
- 原有删除选中、搜索、转班、改名和名单刷新功能不受影响。
