# 上传弹窗 AI 命名设计

## 目标

在前台“上传网页”弹窗的“网页名字”输入框右侧增加“AI命名”按钮。用户选择 HTML 文件或粘贴 HTML 代码后，可在正式上传前让 AI 生成名称并自动填入输入框。

## 交互与布局

- “网页名字”标签下方使用输入框和“AI命名”按钮的横向组合。
- 窄屏时按钮换到输入框下方，不压缩文本可读性。
- 点击后按钮显示“命名中...”并禁止重复点击。
- 成功后将 AI 名称写入“网页名字”，保留用户继续修改的能力，不自动提交上传。
- 弹窗现有消息区显示读取错误、空内容、AI 配置或请求错误和成功提示。

## HTML 输入规则

- 文件和粘贴代码同时存在时，优先读取已选择的 HTML 文件，与现有预览和上传行为一致。
- 未选择文件时，使用“HTML 代码”文本框的内容。
- 文件必须为 `.html` 或 `.htm`；浏览器使用 `File.text()` 读取，不会因 AI 命名而提前上传文件。
- 无 HTML 内容或基本 HTML 结构无效时，不请求 AI。

## 服务端

- 新增 `POST /api/upload-ai-name`，接收 JSON `{ classId, htmlContent }`。
- 接口校验班级存在、当前请求已解锁该班级、班级允许上传、HTML 内容非空且结构有效。
- 接口调用现有 `nameSiteWithLlm`，使用后台已配置的 AI 模型与命名参数。
- AI 名称继续经过现有名称规范化；如命中违禁词，返回明确错误，不填入不可上传的名称。
- 成功返回 `{ title, model }`，不创建项目、不写入项目数据、不生成文件。

## 失败处理

- 班级不存在返回 400，未解锁返回 401，班级已禁止上传返回 403。
- 空内容、非 HTML 文件、HTML 结构错误或违禁词名称在弹窗中显示可读错误。
- AI 配置缺失返回 400，AI 超时或上游错误返回 502。

## 验证

- 服务端测试覆盖成功命名、未解锁、禁止上传、空/无效 HTML 和违禁词结果。
- 前端测试覆盖按钮位置、文件优先、代码回退、加载状态、输入框自动填充和不自动上传。
- 保证现有预览和确定上传流程不变。
