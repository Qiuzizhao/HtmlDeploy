# 技术栈与静态网页部署流程方案

## 1. 技术栈

第一版使用极简技术栈：

```text
Node.js
Express
Multer
本地 JSON 文件
本地文件夹存储
单 HTML 文件前端
```

职责划分：

```text
Node.js
  运行后端服务。

Express
  提供首页、API 接口和 HTML 页面访问路由。

Multer
  处理单个 HTML 文件上传。

data/sites.json
  保存项目列表。

storage/sites/{id}/index.html
  保存用户上传的 HTML 文件。

public/index.html
  平台首页。第一版前端只写这一个文件。
```

## 2. 平台前端实现

平台首页只使用：

```text
public/index.html
```

这个文件里直接包含：

```text
HTML 页面结构
CSS 样式
JavaScript 交互逻辑
上传弹窗
项目卡片列表
iframe 预览大窗口
```

上传控件：

```html
<input id="fileInput" name="file" type="file" accept=".html,text/html" required>
```

页面里的主要前端逻辑：

```text
打开上传弹窗
选择一个 HTML 文件
提交 FormData 到 POST /api/sites
调用 GET /api/sites 获取项目列表
把项目列表渲染成卡片
点击卡片打开大预览窗口
把 iframe.src 设置为项目 url
关闭预览窗口
在新页面打开项目 url
```

## 3. 后端运行方式

Express 服务启动后提供这些路由：

```text
GET  /
GET  /api/sites
POST /api/sites
GET  /site/:id
GET  /site/:id/*
```

说明：

```text
GET /
  返回 public/index.html。

GET /api/sites
  返回 data/sites.json 中的项目列表。

POST /api/sites
  接收网页名字和单个 HTML 文件，保存项目并生成访问地址。

GET /site/:id
  返回 storage/sites/{id}/index.html。

GET /site/:id/*
  当前版本主要用于安全兜底和后续扩展；不存在的文件返回 404。
```

## 4. 部署流程

这里的“部署”是文件上传式部署，不涉及构建、编译或 Git 发布。

流程：

```text
用户打开首页
  ↓
点击上传按钮
  ↓
填写网页名字
  ↓
选择一个 .html 文件
  ↓
浏览器提交 FormData 到 POST /api/sites
  ↓
后端检查文件类型
  ↓
后端生成随机项目 ID
  ↓
后端检查 ID 是否重复
  ↓
后端创建 storage/sites/{id}/
  ↓
后端将上传文件保存为 storage/sites/{id}/index.html
  ↓
后端把项目记录写入 data/sites.json
  ↓
前端刷新项目卡片列表
  ↓
用户点击卡片，iframe 加载 /site/{id}
```

## 5. 文件保存方式

用户上传：

```text
game.html
```

后端生成项目 ID：

```text
abc123
```

保存后的目录：

```text
storage/
  sites/
    abc123/
      index.html
```

项目记录：

```json
[
  {
    "id": "abc123",
    "title": "我的小游戏",
    "createdAt": "2026-06-02T10:00:00.000Z"
  }
]
```

## 6. 项目 ID 生成

项目 ID 用于生成访问地址：

```text
/site/{id}
```

生成规则：

```text
生成随机 ID
  ↓
检查 data/sites.json 中是否已有相同 id
  ↓
检查 storage/sites/{id}/ 是否已经存在
  ↓
如果冲突，重新生成
  ↓
如果不冲突，使用该 ID
```

## 7. 访问与预览流程

首页卡片预览：

```text
点击卡片
  ↓
打开大预览窗口
  ↓
设置 iframe.src = /site/abc123
  ↓
iframe 中显示用户上传的 HTML 网页
```

直接访问：

```text
/site/abc123
```

后端返回：

```text
storage/sites/abc123/index.html
```

## 8. HTML 文件运行方式

服务器只负责保存和返回 HTML 文件。

浏览器访问 `/site/abc123` 后：

```text
服务器返回 index.html
浏览器解析 HTML
浏览器执行 HTML 内部的 CSS 和 JavaScript
```

所以用户上传的 HTML 页面是在访问者浏览器里运行，不是在服务器上运行。

## 9. 文件上传边界

当前版本只支持：

```text
单个 HTML 文件
.html 或 .htm
最大文件大小限制
上传后保存为 index.html
```

当前版本不支持：

```text
多个文件
文件夹上传
zip 解压
单独上传 CSS/JS/图片
服务端构建
```

## 10. 一句话总结

当前技术方案是：

```text
用 Node.js + Express + Multer 做一个极简单 HTML 文件托管 Demo。平台首页只用 public/index.html 一个文件实现。用户上传一个 HTML 文件后，后端保存为 storage/sites/{id}/index.html，记录写入 data/sites.json，首页生成项目卡片，点击卡片后用 iframe 预览 /site/{id}。
```
